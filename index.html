Predicting Swing-and-Miss Probability in MLB Pitching
Using Statcast Data

DSC 148 – Introduction to Data Mining  |  UCSD  |  Spring 2026

Abstract Swinging strikes are among the most decisive events in baseball, directly determining strikeout rates and pitching dominance. This paper presents a data-driven pipeline for predicting whether a given MLB pitch results in a swinging strike, using over 1.46 million pitches from the 2022–2023 seasons sourced from Baseball Savant's Statcast system. We engineer 45 features spanning pitch physics, location, count leverage, pitch tunneling, and sequential pitch context, then compare four classifiers: Logistic Regression, Random Forest, XGBoost, and a PyTorch Neural Network. Under ROC-AUC, the Neural Network achieves the highest discrimination (AUC = 0.760), narrowly edging XGBoost (0.756), while XGBoost achieves the best accuracy (87.8%) by collapsing nearly all predictions to the majority class. Vertical pitch location and induced vertical movement (pfx_z) emerge as the dominant predictors across all models. The severe class imbalance — only 12.3% of pitches result in swinging strikes — remains the central challenge, motivating careful metric selection and future work on threshold tuning and oversampling strategies.

1. Introduction

The swing-and-miss — a batter swinging through a pitch without contact — is a fundamental unit of pitching success. Whiff rate (swinging strikes per swing) is among the best early-indicator statistics for predicting pitcher ERA, and teams increasingly deploy pitch design programs aimed at maximizing this outcome. Understanding which pitch characteristics and game-state factors drive swinging strikes has direct implications for pitcher development, pitch sequencing, and in-game strategy.

Prior work on pitch outcome modeling has relied heavily on logistic regression with handcrafted location features, gradient boosting for expected strikeout modeling, and more recently deep learning approaches for pitch trajectory simulation. However, few published systems integrate pitch tunneling geometry, sequential pitch context, and count leverage into a unified feature set compared across multiple model families.

This project makes three contributions: (1) a fully reproducible Statcast data pipeline covering 1.46 million pitches; (2) a rich 45-dimensional feature set including novel pitch tunneling distance and count leverage index features; and (3) a systematic comparison of four model families with full evaluation under class-imbalance-appropriate metrics.

2. Dataset

2.1 Data Source and Collection

All data was obtained from Baseball Savant via the pybaseball library, which exposes MLB's Statcast pitch-tracking API. We retrieved pitch-level records for the 2022 and 2023 regular seasons (April 2022 – October 2023), yielding 1,464,859 pitches across 29 raw columns after deduplication and cleaning. Statcast tracks ball flight using a combination of Hawk-Eye cameras and radar, capturing position at 30 frames per second with sub-inch accuracy.

2.2 Label Definition

The binary outcome label swinging_strike is set to 1 for pitches with Statcast description codes swinging_strike or swinging_strike_blocked, and 0 for all other outcomes (called strikes, balls, fouls, balls in play). This reflects a decision to model genuine miss outcomes rather than contact-avoidance. The resulting positive rate is 12.26%, a significant class imbalance that we address through class weighting in all models.

2.3 Exploratory Data Analysis

The dataset spans all pitch types, with four-seam fastballs (FF) and sinkers (SI) accounting for the plurality of pitches. Swinging strike rates differ substantially by pitch family: sliders and curveballs generate the highest whiff rates (approximately 18–22%), while four-seam fastballs and sinkers sit near or below the 12.26% average. This motivates including pitch type as a feature.

Vertical pitch location (plate_z) shows a clear non-linear relationship with whiff probability: low pitches near or below the strike zone bottom produce the highest swinging strike rates, consistent with the well-documented phenomenon of batters chasing down-and-away breaking balls. Count state matters substantially — two-strike counts see whiff rates roughly double those of 3-0 counts, motivating our count leverage index feature.

3. Predictive Task

3.1 Task Formulation

We frame the problem as binary classification: given all observable features of a pitch at the moment of release, predict whether the pitch results in a swinging strike. Formally, let x ∈ ℝ46 denote the feature vector for a pitch and y ∈ {0,1} the swinging-strike indicator. We seek a function f: ℝ46 → [0,1] that outputs calibrated swinging-strike probabilities.

3.2 Evaluation Metrics

Given the 12.26% positive rate, accuracy alone is misleading (a trivial all-negative predictor achieves 87.7%). We evaluate using three complementary metrics:

Accuracy = (TP + TN) / (TP + TN + FP + FN) F1 = 2 · Precision · Recall / (Precision + Recall) ROC-AUC = P(score(x⁺) > score(x⁻)) for random positive x⁺, negative x⁻

ROC-AUC is our primary metric because it measures the model's ability to rank positive pitches above negative ones at all possible thresholds, independent of the operating threshold. Average Precision (area under the Precision-Recall curve) provides a complementary imbalance-aware summary.

3.3 Baselines

Logistic Regression serves as the primary linear baseline, assuming swinging-strike log-odds are a linear function of features. This is deliberately interpretable and provides a lower bound on model quality. Random Forest is an ensemble baseline that can capture non-linear interactions without the extensive tuning required by gradient boosting.

3.4 Train/Test Split

We use a stratified random split: 75% training, 10% validation, 15% test, preserving the 12.26% positive rate in each partition. A temporal split (training on 2022, testing on 2023) was also evaluated and produced nearly identical results, confirming generalization across seasons.

4.1 Key Novel Features

Velocity Difference from Average. For each pitch, we compute the deviation of the release speed from that pitcher's season average for that pitch type:

velo_diff = vpitch − μpitcher, type

A pitcher throwing a changeup 5 mph slower than their fastball average is more likely to generate a swinging strike than one with a 2 mph gap, even if the raw velocities are identical. XGBoost ranks this feature 13th overall (importance score 1,042).

Count Leverage Index. We define a count leverage index mapping each (balls, strikes) state to [0,1] based on the empirical swing-rate increase relative to the 0-0 baseline:

leverage(b,s) ∈ {0.0 (0-0), …, 1.0 (3-2)}

Two-strike counts nearly double swinging-strike probability empirically, capturing batter desperation and expanded swing zones.

Pitch Tunneling Distance. We approximate the 3D distance between consecutive pitches at the batter's decision point (~23 ft from the plate), defined as the location where a batter must commit to swinging:

tunnel_dist = √((Δx_tunnel)² + (Δz_tunnel)²)

where x_tunnel and z_tunnel are linearly interpolated positions at the decision point. Pitches that travel nearly the same path to the decision point before breaking apart (low tunnel_dist) are harder for batters to distinguish, increasing whiff probability.

Normalized Plate Location. Raw plate_z does not account for batter height. We normalize by the batter's personal strike zone:

plate_z_norm = (plate_z − sz_bot) / (sz_top − sz_bot)

This makes a pitch at the top of a short batter's zone equivalent to the same relative location for a tall batter, improving cross-batter generalization.

5. Models

5.1 Logistic Regression

Our linear baseline uses L2-regularized logistic regression (C = 0.1) with features scaled by StandardScaler. Class imbalance is addressed via the class_weight="balanced" parameter, which reweights the loss by the inverse class frequency. The decision boundary is:

P(y=1 | x) = σ(wTx + b) = 1 / (1 + e−(wTx+b))

Dominant features are plate_z, plate_z_norm, and plate_x_abs — confirming that pitch location is the strongest linear predictor of swinging strikes.

5.2 Random Forest

We train an ensemble of 300 decision trees with max depth 12 and minimum leaf size 50. The balanced_subsample class weighting strategy applies balanced class weights per bootstrap sample, which is more appropriate than global balancing for ensembles. Feature importance is measured by mean decrease in impurity across all trees.

5.3 XGBoost

XGBoost uses gradient-boosted trees with the hist algorithm for efficiency on large datasets. Key hyperparameters: 500 estimators, max depth 6, learning rate 0.05, 80% subsample and column sample ratios. The model minimizes log-loss with validation-set monitoring. XGBoost's feature importance metric (F-score, number of times a feature is used in splits) differs from impurity-based measures, which explains its different ranking. Notably, pfx_z (vertical movement) is the single most important feature.

5.4 Neural Network

We implement a three-layer MLP in PyTorch with architecture [Input → 256 → 128 → 64 → 1], with Batch Normalization and dropout (p=0.3) after each hidden layer. The output logit is passed through BCEWithLogitsLoss with pos_weight = N_neg/N_pos ≈ 7.15 to correct for class imbalance. Training uses AdamW with cosine learning rate annealing over 30 epochs, batch size 2048.

ℒ = −[y · log σ(ẑ) · pos_weight + (1−y) · log(1 − σ(ẑ))]

Results reveal a striking divergence in model behavior. XGBoost achieves the highest raw accuracy (87.8%) but an F1 of just 0.006 — near-zero — indicating it learned to predict almost no positive cases, exploiting the class imbalance. Its confusion matrix confirms this: of 26,931 actual swinging strikes in the test set, XGBoost correctly identifies only 87. This is a degenerate but technically high-accuracy solution.

In contrast, the Neural Network and Random Forest maintain meaningful recall of swinging strikes (F1 = 0.342 and 0.332 respectively) while achieving strong discrimination (AUC = 0.760 and 0.734). The Neural Network is our recommended model: it leads on the two most meaningful metrics (ROC-AUC and Average Precision) while avoiding the majority-class collapse of XGBoost.

6.2 Confusion Matrix Analysis

The confusion matrices illustrate the accuracy-recall tradeoff clearly. Logistic Regression correctly identifies 17,166 swinging strikes but at the cost of 76,931 false positives. The Neural Network identifies 20,545 swinging strikes with fewer false positives (72,759), achieving the best balance. Random Forest sits between the two, identifying 18,744 swinging strikes. XGBoost's near-total collapse to the majority class (only 87 predicted positive) makes it unsuitable for practical deployment despite leading on accuracy.

6.3 Calibration Analysis

The calibration plot reveals another important finding. XGBoost is well-calibrated at low predicted probabilities (0–0.33 range) — its predictions are reliable where it is confident about non-swinging-strikes. Logistic Regression, Random Forest, and Neural Network are systematically overconfident: when they predict a 70% swinging-strike probability, the empirical rate is closer to 23–30%. This reflects the models inflating probabilities to compensate for class imbalance. Post-hoc calibration via Platt Scaling or Isotonic Regression would improve this for operational deployment.

6.4 Feature Importance

Across all four models, pitch location features (plate_z, plate_z_norm, plate_x_abs) and induced vertical movement (pfx_z) are the top predictors. This is consistent with baseball intuition: breaking balls that drop below the batter's swing path are the primary source of whiffs, and pfx_z captures the magnitude of that drop. Several findings are notable:

Arm-side horizontal location (arm_side_x) ranks highly in Random Forest and Neural Network but not Logistic Regression, suggesting a non-linear interaction: pitches on the arm side of the plate generate more whiffs primarily for breaking pitches, not fastballs, which a linear model cannot capture.

Tunnel distance appears in both XGBoost (rank 15, score 972) and Random Forest (rank 15), validating that pitch tunneling has predictive value beyond raw location — pitchers who set up breaking balls with similar tunnel points as their fastballs generate more swinging strikes.

Previous-pitch velocity ranks in the top 20 for XGBoost (rank 16, score 940) and Random Forest, confirming that sequential pitch effects matter: throwing a changeup after a high-velocity fastball increases swinging-strike probability via the speed differential.

7. Literature Review

7.1 Foundational Pitch Prediction Work

Ganeshapillai & Guttag (2012) established the foundational framework for ML-based pitch prediction at the MIT Sloan Sports Analytics Conference. Using SVMs on PITCHf/x data from the 2008–2009 seasons, they predicted whether a pitcher would throw a fastball or off-speed pitch with 70% accuracy — an 18% average improvement over a naive baseline. Critically, they found that a pitcher's history against the specific batter and their tendency to throw certain pitches in certain counts were the strongest predictors, with count alone being insufficient. This insight directly motivated our count leverage index and previous-pitch sequential features.

7.2 Pitch Type Prediction Using Statcast

Morehouse (2021) extended this line of work to Statcast data, building Random Forest and SVM models for 244 pitchers to predict pitch type (multi-class) using 2017–2018 seasons. Random Forests achieved 51.40% average prediction accuracy, outperforming SVMs (50.52%) and the naive baseline (47.87%). Morehouse found that previous pitch sequence, pitch type tendencies in recent pitches, and pitch count were the top permutation feature importance features — consistent with our findings that sequential context (previous-pitch velocity, previous pitch family) appears in the top 20 features for XGBoost and Random Forest. A key difference is that Morehouse predicts which pitch will be thrown, while our task predicts the outcome — a complementary framing of the same problem.

7.3 Pitch Location and Control Metrics

Ludwig, Brill & Wyner (2025) tackle a closely related problem: measuring pitcher control by separating intended from actual pitch location. Using the same pybaseball/Statcast pipeline as our project, they introduce xCTRL — a metric based on Gaussian Mixture Models fitted per pitcher per pitch type to estimate intended location, then assess execution as the distance between intent and outcome. They find xCTRL is more predictive and stable than existing control metrics (Location+, Command+). Their finding that individual pitcher location tendencies are highly non-uniform motivates our arm_side_x and normalized location features, which adapt to each batter's personal strike zone rather than assuming a single league-wide optimal target zone.

7.4 Deep Learning for Pitcher Performance

Lee & Kim (2025) apply a Temporal Fusion Transformer (TFT) — an attention-based sequence model — to pitcher performance prediction using Statcast data, focusing on ERA prediction. Their work demonstrates that capturing temporal dependencies across pitch sequences with deep sequential models provides meaningful gains over non-sequential approaches. This directly connects to our key limitation: our Neural Network treats each pitch largely independently (aside from single previous-pitch features), while a full sequence model over at-bats could capture the multi-pitch setups that generate swinging strikes. The TFT architecture they propose represents a natural direction for future extensions of our work.

7.5 Position Among Related Work

Our work sits at the intersection of these threads: we use Statcast data at the same scale as Ludwig et al. (2025) and Morehouse (2021), apply the multi-model comparison framework of Ganeshapillai & Guttag (2012), and identify the limitation of non-sequential modeling that Lee & Kim (2025) address with the TFT. The primary novelty of our contribution is the focus on swing-and-miss probability specifically (rather than pitch type or ERA), combined with pitch tunneling geometry and count leverage as novel features not present in prior pitch-level ML work.

8. Discussion

8.1 Why XGBoost Collapsed

XGBoost's near-zero F1 despite leading on accuracy is a textbook class-imbalance failure. The model found that predicting all negatives yields 87.7% accuracy, which minimizes log-loss more efficiently than learning a complex decision boundary for the 12% minority class. Our class weighting was insufficient to overcome this at the default 0.5 decision threshold. This could be remedied by threshold tuning (e.g., setting threshold to 0.15) or by using scale_pos_weight = N_neg/N_pos ≈ 7.15 in XGBoost's parameters — a hyperparameter we did not grid-search.

8.2 Why the Neural Network Wins

The Neural Network's top AUC (0.760) and F1 (0.342) stem from its ability to learn non-linear interactions between features — particularly between movement (pfx_z) and location (plate_z), and between arm-side location and pitch type. Batch normalization helps the model handle the heterogeneous feature scales (velocity in mph, spin in rpm, location in feet) without separate scaling per feature type. The BCEWithLogitsLoss pos_weight of 7.15 effectively upsamples the minority class in gradient computation.

8.3 Limitations

Our model treats each pitch independently except for the previous-pitch features. Full at-bat sequencing — which pitcher-batter matchups within at-bats generate the most whiff-inducing setups — is not captured. A recurrent model (LSTM or Transformer over pitch sequences) would better represent this structure. Additionally, we do not model pitcher or batter identity directly (beyond handedness), missing player-specific tendencies like a batter's historically poor performance against breaking balls on the outer third.

9. Conclusion

We presented a comprehensive swing-and-miss prediction pipeline using 1.46 million Statcast pitches, 46 engineered features, and four model families. Our key findings are:

(1) Vertical pitch location and induced vertical movement (pfx_z) are the dominant predictors of swinging strikes, consistent across all model families and with established baseball knowledge.

(2) The Neural Network achieves the best practical performance (AUC = 0.760, F1 = 0.342, AP = 0.288), avoiding the majority-class collapse that afflicts XGBoost at the default threshold.

(3) Pitch tunneling distance and velocity differential from pitcher average provide measurable predictive value beyond raw physics features, validating the theoretical importance of pitch deception in generating swinging strikes.

(4) Class imbalance (12.26% positive rate) is the central methodological challenge. Future work should explore SMOTE oversampling, threshold optimization, and sequence-aware models that capture full at-bat context.
